VERIFYRE - SETUP GUIDE (XAMPP / htdocs)
=========================================
Read this fully before copying files. Each step matters.

----------------------------------------------------
STEP 0: Why you can't just drop this folder into htdocs
----------------------------------------------------
This ZIP only contains the files a developer writes by hand: routes,
controllers, models, Blade views, CSS/JS, and a sample .env. It does
NOT contain the Laravel framework itself (the "vendor" folder, the
"artisan" file, "bootstrap/app.php", "config/*.php", etc.) - those
are auto-generated by Composer and are version-specific, so they
have to be generated fresh on your machine, then my files placed
on top.

----------------------------------------------------
STEP 1: Install Composer (if you don't have it)
----------------------------------------------------
Download and install from https://getcomposer.org/download/
Confirm it worked by opening Command Prompt and running:
    composer --version

----------------------------------------------------
STEP 2: Generate a fresh Laravel skeleton inside htdocs
----------------------------------------------------
Open Command Prompt, go to your XAMPP htdocs folder, then run:

    cd C:\xampp\htdocs
    composer create-project laravel/laravel Verifyre_Laravel

This creates C:\xampp\htdocs\Verifyre_Laravel with the full framework.

----------------------------------------------------
STEP 3: Copy my files into that folder
----------------------------------------------------
From this ZIP, copy the following INTO C:\xampp\htdocs\Verifyre_Laravel,
overwriting/merging as needed:

  routes/web.php                  -> overwrite
  app/Http/Controllers/*          -> copy in (new files)
  app/Http/Middleware/*           -> copy in (new file)
  app/Models/*                    -> copy in (new files)
  app/Mail/*                      -> copy in (new folder)
  resources/views/*               -> copy in (merges with existing views folder)
  public/css/*                    -> copy in
  public/js/*                     -> copy in
  public/images/*                 -> copy in

Do NOT copy my .env.example over the real .env that Laravel generated.
Instead, OPEN both files and copy these specific lines from my
.env.example into your project's real .env, replacing the matching
lines:

  DB_DATABASE=verifyre_laravel_database
  DB_USERNAME=root
  DB_PASSWORD=
  MAIL_MAILER=smtp
  MAIL_HOST=smtp.gmail.com
  MAIL_PORT=587
  MAIL_USERNAME=your_email_here@gmail.com
  MAIL_PASSWORD=your_16_char_app_password_here
  MAIL_ENCRYPTION=tls
  MAIL_FROM_ADDRESS="your_email_here@gmail.com"
  MAIL_FROM_NAME="VeriFyre"
  SITE_CONTACT_NUMBER="+63 900 000 0000"
  SITE_CONTACT_EMAIL="innovision.verifyre@gmail.com"

Then fill in YOUR real Gmail address and an "App Password" (not your
normal Gmail password) for MAIL_USERNAME / MAIL_PASSWORD / MAIL_FROM_ADDRESS:
  1. Turn on 2-Step Verification on your Google account.
  2. Go to https://myaccount.google.com/apppasswords
  3. Generate a password for "Mail" and paste the 16-character code
     into MAIL_PASSWORD (remove spaces).
This is what makes the OTP emails actually send.

----------------------------------------------------
STEP 4: Set up the database
----------------------------------------------------
1. Start Apache and MySQL in the XAMPP control panel.
2. Open http://localhost/phpmyadmin
3. Click "Import" (or "SQL" tab), then run/import the file
   verifyre_laravel_database.sql (provided separately in chat).
   It creates the database and all tables automatically.

----------------------------------------------------
STEP 5: Final commands (Command Prompt, inside the project folder)
----------------------------------------------------
    cd C:\xampp\htdocs\Verifyre_Laravel
    php artisan key:generate
    php artisan storage:link

`storage:link` is required so uploaded fire-report photos are
viewable in the browser.

----------------------------------------------------
STEP 6: Run it
----------------------------------------------------
Easiest option (recommended for development):
    php artisan serve
Then visit: http://127.0.0.1:8000

If you specifically want it served through XAMPP's Apache at a
htdocs-style URL, visit:
    http://localhost/Verifyre_Laravel/public/
(Laravel's entry point is always the "public" folder - this is
normal for every Laravel project, not specific to ours.)

WHAT'S WORKING RIGHT NOW
----------------------------------------------------
- All pages from your Figma: Home, Report a Fire, Success, Education,
  Cybersecurity, About Us, Login, Register, Forgot Password, Admin
  Home/Activity/Reports, Firefighter Home/Reports/Profile.
- Collapsible mobile navbar + Contact Us footer on every page.
- GPS permission request + mini Leaflet map on Home.
- Camera capture flow on Report a Fire (asks permission, lets you
  snap and retake a photo).
- tel:911 auto-trigger on the Report and Success pages (opens your
  phone's own dialer automatically - the final "place this call?"
  confirmation is controlled by the phone's OS and can't be skipped
  by any website, for your own safety).
- Crowdsourcing popup UI + "Verify" button + camera + submission flow
  on the Home page (the real "is someone nearby an active report?"
  proximity check still needs the backend - see note in
  public/js/crowdsourcing.js).
- Full email OTP flow for Register and Forgot Password - this
  actually sends real emails once you fill in MAIL_* in .env.
- Admin/Firefighter dashboard pages are viewable by anyone right now
  (no login wall yet) so you can review the UI. RoleMiddleware.php is
  already written and commented with exactly how to wire it in for
  the backend/RBAC phase.
